## 단기 작업 (P0 — 즉시)

### P0-1. iOS 통화 후 요약 배너 (MISSING)

**상태**: 출시 코드에 없음. 향후 작업 필요.

**문제**:
- 현재 iOS는 PRECALL 시점에 Live Activity만 표시 (Pro 모델 한정)
- 통화 종료 후 "방금 받은 통화가 인증된 가족이었음" 확인 불가

**작업**:
- 통화 종료 직후 NSE를 통한 요약 알림 발송
- 푸시 내용: "{name}님과의 인증 통화 종료"
- 탭 시 해당 연락처 call-shield 페이지로 이동

**우선순위**: 높음 (iOS 사용자 경험 개선)

### P0-2. 서버 이상 탐지 — 다중 통화 공격 버스트 (MISSING)

**상태**: 출시 코드에 없음.

**문제**:
- 짧은 시간 내 동일 발신자/수신자 다중 통화 시 별도 탐지 없음
- 공격자가 PRECALL 게이트를 통과한 후 짧은 시간 내 여러 통화 시 일반 트래픽으로 처리

**작업**:
- VP API에 시계열 이상 탐지 로직 추가
- 임계값: 동일 페어 1분 내 3회 초과 통화 → 경고 로그 + 사용자 알림
- 임계값: 동일 발신자 5분 내 다수 수신자에게 통화 → 발신자 차단 검토
- `vp_audit_log` 활용

**우선순위**: 중간 (방어 강화)

## 단기 작업 (P1 — 출시 준비)

### P1-1. 기능 안정화

PDF에서 ACTIVE로 표시된 모든 기능의 **stg → prod 전환 준비**.

**작업**:
- E2E 테스트 커버리지 95%+
- 부하 테스트 (PRECALL 1만 RPS)
- 장애 시나리오 테스트:
  - VP API 다운 시 Main Backend 동작
  - FCM/APNs 지연 시 동작
  - Redis 다운 시 LIVENESS 팬아웃 동작
- 보안 감사 (외부 펜테스팅)

### P1-2. 운영 모니터링

**작업**:
- 메트릭 대시보드:
  - PRECALL 성공률 / 실패율 (4-check별)
  - 페어링 요청 → CONFIRMED 전환율
  - 하트비트 미수신 단말 수
  - SIM-SWAP 탐지 건수 (월별)
- 알람:
  - PRECALL 실패율 > 1% → 페이지
  - 하트비트 미수신 > 10% → 경고
  - 서버 침해 시도 패턴 → 즉시 알림

### P1-3. 사용자 온보딩 개선

**작업**:
- 첫 페어링 가이드 (가족 1명과 페어링 권장)
- TTS 옵트인 권장 모달 (고령 사용자)
- 새 단말 알림의 명확한 CTA

## 중기 작업 (P2 — 3~6개월)

### P2-1. PushKit-VoIP 위장 (NOT FEASIBLE → 재평가)

**상태**: PDF에서 NOT FEASIBLE로 분류.

**재평가 사유**:
- Apple 정책이 향후 변경될 수 있음
- VoIP 푸시는 통화별 코드 실행 가능 (단, "실제 VoIP 통화"만 허용)
- 정책 변경 또는 우회 방법 발견 시 즉시 적용

**작업** (정책 허용 시):
- VoIP 푸시로 통화별 상태 표시
- iOS도 Android와 동등한 통화별 시각/음성 안내

### P2-2. 포그라운드 iOS 통화별 TTS (DEFERRED → 재평가)

**상태**: PDF에서 DEFERRED.

**작업** (가능성 평가 후):
- iOS 앱이 포그라운드일 때만 TTS 발화
- 비-Pro 모델 사용자 대안

### P2-3. 다국어 지원

**작업**:
- 영어, 중국어, 일본어 음성 안내
- 한국 거주 외국인 대상

### P2-4. 사업자 디렉토리 확장

**상태**: `vp_org_directory` 테이블은 있으나 확장 필요.

**작업**:
- 은행, 통신사, 정부 기관 등 주요 사업자 번호 등록
- 사업자 번호로부터의 통화는 "공식 사업자" 라벨 표시
- 사업자 사칭 보이스피싱 (예: "OO은행입니다") 추가 차단
