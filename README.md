## 🎯 v7 변경 사항 요약

### 1. PDF 기준 4-Component 아키텍처 정식화

이전 문서들의 컴포넌트 명칭 / 책임 / DB 분리 / JWT 경계를 PDF 출시 코드 기준으로 통일:

- **Mobile App** (Flutter)
- **Main Backend** (Spring Boot 3.2 + `chondb`)
- **VP API** (NestJS + `chon_dev`/`chon_stg`)
- **FCM + APNs** (Push)

### 2. 신뢰 체인의 정밀 명세

- **하드웨어 비공개 키 절대 미반출** 원칙 강조
- `trust_state` 3-state: TRUSTED / PENDING / REVOKED
- 4시간 하트비트 메커니즘
- App Attest (iOS) / Play Integrity (Android) 검증 체인

### 3. PRECALL 4-Check 게이트

PRECALL 신뢰 게이트의 4가지 검사를 정식 명세화:
1. `trust_state == TRUSTED`
2. `last_heartbeat_at` 신선
3. 서명 검증 (저장된 공개 키)
4. 관계 게이트 (family-tree ∪ prior-CONFIRMED)

### 4. LIVENESS_UPDATE 대비책 명세

iPhone 시스템 다이얼러 제약을 우회하는 대비책 메커니즘:
- TRUSTED 전용 게이트
- 페어당 10분 스로틀
- TTL 15분
- Redis 사용자별 60초 슬롯

### 5. 3-State 화면 표시

- **VERIFIED** (🟢 녹색) — 하드웨어 검증된 페어
- **UNKNOWN** (🟡 노란색) — Pending 또는 liveness 전용
- **SIM-SWAP** (🔴 빨간색) — 알려진 페어, 칩 없음

### 6. 12가지 공격 시나리오 매트릭스

A부터 L까지 12개 시나리오의 방어 메커니즘 정식화. **모두 ACTIVE (출시 코드 반영)**.

### 7. 데이터 모델 정밀 명세

폰 보안 저장소 / `chondb` / `chon_dev`의 모든 테이블 스키마와 용도 명세.

### 8. 출시 현황의 정확한 기록

- **✅ ACTIVE**: 21개 기능 (출시 코드 반영)
- **⏳ MISSING**: 2개 기능 (iOS 통화 후 요약, 서버 이상 탐지)
- **❌ NOT FEASIBLE / DEFERRED**: 2개 (PushKit-VoIP, iOS 통화별 TTS)

## 🔑 핵심 메시지

> **CHON은 번호를 신뢰하지 않습니다. CHON은 하드웨어(칩)를 신뢰합니다.**
>
> SIM-swap, 번호 스푸핑, Caller-ID 주입, 비밀번호 탈취, FCM 가로채기 — 그 어느 것도 CHON을 무력화할 수 없습니다.
> 왜냐하면 그 어느 것도 피해자의 칩을 주지 않기 때문입니다.
