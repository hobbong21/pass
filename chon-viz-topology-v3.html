🎨 CHON 시각화 토폴로지 v3 — 정렬 원칙 일반화

출생 순서 정렬 원칙을 가족 → 종친회 → 조직도로 확장.

한 페이지에서 3개 토폴로지를 토글하며 동일한 정렬 규칙이 어떻게 적용되는지 확인하실 수 있습니다. 모든 토폴로지가 sortOrder 필드 하나로 통일.

v3 정렬 일반화 Clan (종친회) Org (조직) Family (가족)

🎯 통일된 정렬 원칙

같은 부모(상위 노드)를 공유하는 노드들 = 형제 그룹. 형제 그룹은 항상 sortOrder 오름차순으로 왼쪽 → 오른쪽 정렬됩니다.

토폴로지 "형제 그룹" 정의 정렬 기준 좌측 = 우선순위
👨‍👩‍👧 Family 같은 부모의 자녀 birthOrder 맏이 → 막내
🏛 Clan (종친회) 같은 시조의 자손 (항렬별) birthOrder (항렬 내) 맏이 → 막내
🏢 Org (조직) 같은 상사의 부하 rank → hireDate 상위직 / 선임자 → 후임
// 통일된 정렬 함수 — 모든 토폴로지에서 재사용 function sortSiblings(group, topology) { return [...group].sort((a, b) => { if (topology === 'org') { // 1) 직급 순서, 2) 입사일 순서 const rankDiff = (a.rankLevel || 99) - (b.rankLevel || 99); if (rankDiff !== 0) return rankDiff; return (a.hireDate || '9999').localeCompare(b.hireDate || '9999'); } // family / clan: birthOrder만 사용 return (a.sortOrder || 999) - (b.sortOrder || 999); }); }
👨‍👩‍👧 Family — 출생 순서 정렬
아버지가 셋째, 나도 셋째인 경우 예시. 같은 부모의 자녀들은 출생 순서대로 좌→우 정렬됩니다.
📐
가족 정렬 규칙 같은 부모(parentId가 같은) 노드들 = 형제 그룹 → birthOrder 오름차순 좌→우 정렬
예: 큰아버지(1) → 작은아버지(2) → 아버지(3) → 고모(4)
🏛 Clan — 항렬 × 출생 순서
김씨 종친회 예시. 항렬(세대)이 같은 사람들끼리 모이며, 각 항렬 내에서 다시 출생 순서로 정렬.
항렬 = 같은 세대의 종친에게 사용되는 한자 (예: 광(光)·진(振)·수(秀)).
📐
종친회 정렬 규칙 (2단계) generation (항렬) 그룹화 → 각 항렬 내 birthOrder로 좌→우 정렬
예: 광(光) 항렬 → 김광수(1), 김광호(2), 김광민(3) | 진(振) 항렬 → 김진수(1), 김진호(2)
🏢 Org — 직급 × 입사일 정렬
개발팀 예시. 같은 상사를 가진 동료들은 직급 순서 우선, 같은 직급 내에서는 입사일 순서로 정렬.
📐
조직 정렬 규칙 (2단계) 같은 managerId 그룹화 → 1차 rankLevel (높을수록 왼쪽), 2차 hireDate (이를수록 왼쪽)
예: 시니어(2022) → 시니어(2023) → 미들(2021) → 미들(2024) → 주니어(2024)

💾 통일된 데이터 모델

모든 토폴로지의 노드가 공통 인터페이스를 따릅니다. 토폴로지별 추가 필드만 다름:

interface SortableNode { // 공통 필드 (모든 토폴로지) id: string; name: string; parentId?: string; // 상위 노드 (이를 공유하면 형제) sortOrder: number; // 형제 그룹 내 정렬 (1, 2, 3...) // Family birthOrder?: number; // = sortOrder 별명 // Clan (종친회) generation?: number; // 항렬 (위에서부터 0, 1, 2...) generationName?: string; // "광", "진" 등 항렬자 // Org rankLevel?: number; // 직급 (1=대표, 2=임원, 3=팀장, 4=시니어...) rankName?: string; // "팀장", "시니어 개발자" 등 hireDate?: string; // ISO date (입사일) managerId?: string; // = parentId 별명 }

핵심 통찰: 가족의 birthOrder, 종친회의 generation+birthOrder, 조직의 rankLevel+hireDate는 모두 "같은 상위 노드 아래에서 어떻게 형제들을 정렬할 것인가"라는 동일한 문제의 답입니다.

🔄 토폴로지 간 변환 일관성

같은 사람이 여러 토폴로지에 속할 때 정렬 규칙은 각각 독립적으로 적용됩니다:

  • 김광수 (가족에선 부모의 셋째 자녀) ← birthOrder=3
  • 김광수 (종친회에선 광 항렬의 둘째) ← generation+birthOrder=(2,2)
  • 김광수 (회사에선 팀장) ← rankLevel=3

→ 한 사람의 sortOrder가 토폴로지별로 독립이므로, 같은 노드가 다른 그래프에서 다른 위치에 표시됩니다. 자연스러운 컨텍스트 적응.
